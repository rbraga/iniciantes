Desenvolvimento

Clonar um Projeto

Abrir o Git-Extensions. Selecionar Clone repository.

Informe a Url do repositório, local onde o arquivo será salvo em sua máquina e o nome da pasta onde será salvo o repositório.

Após clonar o repositório, será exibida uma mensagem para abrir o repositório, não é necessário abrir o projeto agora, vamos abrir o projeto no Visual Studio.

Commit Local

Para realizar o commit no branch “develop”, após realizar suas alterações clique no botão Commit (develop). note que entre parênteses está escrito o nome do branch em que estamos trabalhando.

Na janela de commit, selecione os arquivos que deseja enviar no commit, e selecione a opção desejada.

  • Commit : Realiza um commit local com os arquivos selecionados.
  • Commit & push:  Realiza um commit local e em seguida envia o commit ao repositório remoto.
  • Reset all changes:  Desfaz todas as alterações e todos os arquivos alterados, inclusive os que não estão selecionados.
  • Amend Commit:  O commit será mesclado ao commit anterior, formando um só commit, é ideal ao realizar uma alteração, e logo após mais alterações, que deveriam estar no commit anterior.

Após selecionar os arquivos desejados para o commit, clique em Commit .

No commit foi selecionado o Arquivo “README.md”.

Se seu commit for realizado com sucesso, uma janela parecida com essa deve ser exibida.

Atualizar repositório local (Pull)

Quando precisar atualizar seu repositório local com o repositório remoto.

Clique sobre o botão “Pull”.

Na janela de Pull, informamos a origem de onde será atualizado nosso repositório, no nosso caso o origin que é nosso repositório remoto, marcamos a primeira opção de merge “ Merge remote branch into current branch ”.

Enviar commit locais para o repositório remoto (Push)

Clique sobre o botão “Push”.

Uma janela com as opções para pull será mostrada, deixamos o remote no origin, que é nosso repositório remoto, e deixamos na aba Push branches, vamos somente atualizar o branch “develop” remoto com nossas alterações e clique em “Push”.

Criando novas funcionalidades (Feature)

Para realizar grandes alterações no projetos, o ideal é trabalhar em um branch separado.

Vamos criar um branch a partir do branch “develop”.

Primeiro faça um checkout no branch “develop”, vá em “Git > Checkout branch”, marque “Local branch”, selecione “develop” em “Select branch” e deixe a opção “Don’t change” marcada.

Após o checkout, crie um branch em “Git > Create branch”, como acabamos de fazer um checkout no branch develop não é necessário alterar a primeira opção. Em “Branch name” informe um nome para sua nova funcionalidade.

Note que agora o botão “Commit” aparece apontando para o branch da nossa nova funcionalidade, todos os commits serão realizados nesse branch.

Após finalizar sua nova funcionalidade, você precisa unir ela ao branch de desenvolvimento.

Primeiro vamos voltar para o branch de desenvolvimento, fazendo um checkout. Selecione “Git > Checkout branch” e selecione novamente o branch develop.

Agora vamos fazer um merge. Selecione “Git > Merge branches”. O current branch deve estar no “develop”, selecione em “Merge with” o branch da nova funcionalidade, deixe em “Keep a single branch line” e clique em Merge.

Após o merge, sua nova funcionalidade já está no branch develop, agora podemos atualizar o repositório remoto com essa nova funcionalidade, para isso faça um “Pull” e um “Push”.

Corrigir um erro

Em determinados momentos, precisamos realizar uma correções de um erro que está na ultima versão, mas como essa versão será lançada somente para corrigir erros, não vamos fazer essa correção no branch develop, que é usado para desenvolvimento.

Primeiro faça um checkout no branch remoto “master”, selecione “Git > Checkout branch” e selecione o branch “origin/master”.

Crie um branch com o nome do erro a ser corrigido, vá em “Git > Create branch”.

Após criar o branch, realize a correção e commit local. Precisamos atualizar o master com essa correção, faça um checkout no branch master. Selecione “Git > Checkout” conforme a imagem.

Faça um merge com o branch onde foi corrigido o erro.

O mesmo também necessita ser replicado no ambiente de desenvolvimento. Para isso vamos fazer o mesmo processo no branch develop. Faça um checkout no branch develop.

E em seguida um merge.

Para finalizar, o processo faça um pull e um push no repositório remoto dos branchs master e develop.

Resolver conflitos

Em alguns casos, durante o desenvolvimento podem ocorrer conflitos. Isso ocorre quando dois desenvolvedores fazem alterações na mesma linha de um mesmo arquivo.

Vamos supor que o ao enviar minhas alterações, o git extensions deverá mostra uma janela informando que minhas alterações foram rejeitadas.

Vamos atualizar nosso projeto com um merge escolhendo a terceira opção “ Pull with merge ”.

Após selecionar a opção o git mostra outra janela com as informações, se o merge foi realizado, no nosso caso o arquivo “README.md” possui conflitos.

Clique em OK, e uma janela deverá perguntar se deseja resolver o conflito, selecione “Sim”.

Na janela selecione o arquivo que deseja resolver o conflito e selecione “Open in kdiff3”.

Ao abrir o Kdiff3, surge uma janela com informações sobre o conflito, ocorreram 4 conflitos, sendo que 3 deles o git já resolveu automaticamente.

 

O Kdiff3 mostra os 4 conflitos, e já vem selecionado o que precisamos resolver, na parte superior existem areas (A,B,C). A area A mostra como a linha estava antes do conflito, e as areas B e C, as alterações que foram realizadas. Selecione a que for a melhor solução, se existirem mais conflitos não resolvidos navegue usando as setas ao lado.

Após resolver o conflito, nosso arquivo ficou com dessa forma. Para finalizar, salve e feche o Kdiff3.        

Após resolver esse conflito vamos realizar um commit.

E novamente realizamos um pull no repositório remoto.
